# PyAutoGUI-Customed GUI化 設計

## 背景

現状は単一の `main.py` による CLI スクリプト。画面内から `assets/target.jpg` を探し、見つかればその中心をクリックし続ける無限ループ。設定（`CONFIDENCE`）はコードを直接編集して変更する必要があり、開始/停止もターミナル操作やフェイルセーフ頼みになっている。

これをデスクトップ GUI 化し、ボタン操作・画像選択・設定調整・実行状況の可視化を可能にする。

## 目的（解決したいこと）

- 開始/停止をボタンで安全に行う
- 対象画像を GUI（ファイル選択）で指定する
- CONFIDENCE・待機時間・グレースケール検出を GUI で調整する
- 検出の成否・エラーなど実行状況をリアルタイムに可視化する

## 技術選定

- **GUI フレームワーク**: PySide6 / Qt
- **自動操作**: 既存どおり `pyautogui`（`opencv-python` による confidence マッチ）
- **言語/管理**: Python 3.14 / uv（既存踏襲）

PyAutoGUI は GUI と同一 PC 上でデスクトップを操作するため、Web ではなくネイティブのデスクトップ GUI とする。

## アーキテクチャ

自動操作ループはワーカースレッド（QThread）で実行し、GUI スレッドと分離する。`locateCenterOnScreen` は画面全体走査で数百 ms〜秒かかることがあり、GUI スレッドで回すと UI が固まるため。

代替案として GUI スレッド上で `QTimer` による疑似ループも検討したが、上記の理由で不採用。

### ファイル構成

現状の単一 `main.py` を役割ごとに分割する。

```
main.py        … エントリポイント（QApplication 起動のみ）
gui.py         … メインウィンドウ（操作UI・設定・ログ表示）
worker.py      … 自動操作ループ（QThreadベース、画像検出→クリック→待機→移動）
settings.py    … 設定値を集約する dataclass（AutomationSettings）
```

### 各ユニットの責務

- **settings.py / `AutomationSettings`**: 画像パス・confidence・待機秒・グレースケールを保持する dataclass。GUI と worker が共有する単一の設定モデル。将来の設定永続化（QSettings 等）に備え、設定の出入り口をこのオブジェクトに一本化する（後述）。
- **worker.py**: `AutomationSettings` を受け取り、ループで検出＆クリック。状態（検出成功/失敗/エラー、移動）を Qt シグナルで通知。停止要求を受けてループを安全に終了する。GUI には依存しない（pyautogui のみに依存）。
- **gui.py**: ウィジェット配置とユーザー入力。入力値を `AutomationSettings` に反映し、開始で worker を起動、停止で worker に停止要求。worker のシグナルを受けてログ・ステータスを更新する。
- **main.py**: `QApplication` を起動し、メインウィンドウを表示するのみ。

## UI 構成（1 ウィンドウ）

- **対象画像**: パス表示 ＋「参照」ボタン（ファイル選択ダイアログ）
- **CONFIDENCE**: スライダー 0.1〜1.0（初期値 0.8、推奨値 0.8 をラベルに明記）
- **待機時間**: 数値入力（秒、初期値 4）
- **グレースケール検出**: チェックボックス（初期 ON）
- **開始 / 停止**: ボタン（実行中は開始を無効化、停止のみ有効）
- **ステータス表示**: 実行中 / 停止中、最後の検出結果
- **ログ**: 検出成功・失敗・エラーを時刻付きで追記表示

## 自動操作ループの挙動

旧スクリプトのループ順序を維持する。

1. 画面内から対象画像を探す
2. 見つかればその中心をクリック、なければログに記録
3. 設定秒数だけ待機
4. マウスを画面右下から 100px 内側へ移動（**ホバー解除のため必須**。移動しないと対象のホバー状態が継続し、見た目が変わって次回検出できなくなる）
5. 1 へ戻る

ステップ 4 の移動は GUI の設定項目には出さず、固定挙動とする（廃止しない）。

## 安全停止

- GUI の「停止」ボタンでループを安全に終了する（停止要求フラグをワーカーが各ループ先頭で確認）。
- PyAutoGUI のフェイルセーフ（マウスを画面四隅へ勢いよく動かす）は **ON 維持**。暴走時の保険として残す。

## 将来拡張：設定の永続化（今回は実装しない）

前回使用した画像パス・設定値を次回起動時に復元する機能は、今回は YAGNI で見送る。ただし後から最小差分で追加できるよう設計する。

- 設定値はすべて `AutomationSettings`（dataclass）に集約する。
- GUI は個別ウィジェットの値を直接散在させず、「ウィジェット ⇄ `AutomationSettings`」の相互変換メソッド経由で読み書きする。
- これにより将来、起動時に `AutomationSettings` をデシリアライズしてウィジェットへ反映、終了時にシリアライズして保存、という形（QSettings 等）で永続化を追加できる。今回の実装ではシリアライズ層は作らない。

## テスト

- **worker.py**: `pyautogui` をモックし、検出ロジックをユニットテスト（検出成功でクリックが呼ばれる / 検出失敗で失敗シグナル / 例外時にエラーシグナル / 停止要求でループ終了）。
- **settings.py**: ウィジェット値との相互変換ロジックがあれば対象に含める。
- **gui.py**: GUI 層は手動確認を中心とする。

## 非対象（スコープ外）

- 画面範囲のドラッグキャプチャによる画像指定（ファイル選択のみ）
- 設定の永続化の実装（設計上の準備のみ）
- 複数対象画像の同時検出
