# py-pyautogui

PyAutoGUI を使った画面自動操作ツール。GUIウィンドウで対象画像・パラメータを設定し、画面上から指定した画像を探し出して自動でクリックし続けます。

## 必要環境

- Python 3.14（`.python-version` で指定）
- パッケージ管理: [uv](https://docs.astral.sh/uv/)

## 依存パッケージ

| パッケージ | 役割 |
| --- | --- |
| `pyautogui` | マウス操作・画面キャプチャ・画像検索 |
| `opencv-python` | `confidence`（曖昧一致）による画像マッチの精度向上 |
| `pillow` | 画面キャプチャの内部処理（`pyscreeze` が要求） |
| `PySide6` | GUIウィンドウの描画 |

> **補足**: `pyscreeze` は Python 3.12 以降向けに Pillow 依存をメタデータ宣言していないため、Pillow を明示的に追加しています。

## セットアップ

```powershell
# 依存をインストール（初回のみ）
uv sync
```

## 使い方

起動方法は2通りあります。

**方法 1: `run.bat` をダブルクリック**（Windows 推奨）

エクスプローラーで `run.bat` をダブルクリックするとGUIウィンドウが開きます。

**方法 2: コマンドで起動**

```powershell
uv run main.py
```

どちらもGUIウィンドウが開きます。

### 操作手順

1. 「**参照**」ボタンをクリックし、画面上で探したい対象のスクリーンショット（`.jpg` / `.png` など）を選択する
2. 必要に応じてパラメータをGUIで調整する（後述）
3. 「**開始**」ボタンをクリックすると自動化が始まる
4. 「**停止**」ボタンをクリックするとループが止まる

### 動作の流れ

「開始」後、以下を繰り返します。

1. `locateCenterOnScreen` で画面内から対象画像を探す
2. 見つかればその中心をクリック（見つからなければログに記録）
3. 設定した待機時間だけ待つ
4. マウスを画面右下から 100px 内側へ移動する（ホバー解除のための固定挙動）
5. 1 へ戻る

## チューニング

GUIウィンドウ上のコントロールでパラメータを調整できます。

| 設定項目 | 調整方法 | 説明 |
| --- | --- | --- |
| CONFIDENCE | スライダー / 数値入力 | 画像マッチの厳しさ（`1.0` = 完全一致） |
| 待機時間（秒） | 数値入力 | 1ループあたりの待機秒数 |
| グレースケール | チェックボックス | ON にすると色を無視してマッチ（高速・軽量） |

| 症状 | 対処 |
| --- | --- |
| 検出漏れ（表示されているのに見つからない） | CONFIDENCE を `0.7` 程度へ下げる |
| 誤検出（別の場所をクリックする） | CONFIDENCE を `0.85`〜`0.9` へ上げる |
| それでも不安定 | 対象画像を PNG で撮り直す（JPEG はノイズが乗るため） |

## 緊急停止（フェイルセーフ）

停止方法は2つあります。

- **GUIの「停止」ボタン**: クリックするとループが安全に終了します
- **フェイルセーフ**: マウスを画面の四隅（特に左上）へ勢いよく動かすと、PyAutoGUI のフェイルセーフが作動してスクリプトが停止します

フェイルセーフは常に ON です（無効にする操作はありません）。暴走時の最終手段として覚えておいてください。

## 注意

- `assets/` は `.gitignore` 対象です。対象画像はローカル専用で、リポジトリにはコミットされません。
- 対象画像が未選択の状態で「開始」を押すとログに「対象画像が未選択です」と表示され、起動しません。
